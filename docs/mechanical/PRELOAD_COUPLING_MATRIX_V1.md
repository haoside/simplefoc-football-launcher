# PRELOAD_COUPLING_MATRIX_V1

## 目的
把 `包胶厚度` 与 `压紧量` 的联动从口头判断改成显式表格，避免 `v1.2` 阶段在未校核情况下锁死错误组合。

当前冻结参数：
- `inner_d = 232`
- `motor_d = 63`
- `contact_sink = 2.0 ~ 3.0 mm`
- 默认 `contact_sink = 2.5 mm`
- `rubber_sleeve_t = 1.5 mm`（当前 CAD 名义值）

---

## 一阶估算口径
在当前 CAD 表达里，可先按下面的一阶估算判断风险：

```text
理论有效压紧 ≈ contact_sink + rubber_sleeve_t
```

说明：
- `contact_sink` 表达的是电机工作带向球道侵入的基准量
- 若外转子外表面再包一层胶，接触外径会继续增大
- 真实压紧量仍要靠实物装配复核，这里只是 v1.2 阶段的先行筛选表

---

## 联动矩阵（单位：mm）

| `rubber_sleeve_t` \ `contact_sink` | 2.0 | 2.5 | 3.0 |
|---|---:|---:|---:|
| 0.0 | 2.0 | 2.5 | 3.0 |
| 1.0 | 3.0 | 3.5 | 4.0 |
| 1.5 | 3.5 | 4.0 | 4.5 |
| 2.0 | 4.0 | 4.5 | 5.0 |

---

## 当前工程判断
### 可直接落在目标区间 `2–3mm` 的组合
只有这些组合一阶上是安全的：
- `rubber_sleeve_t = 0.0` + `contact_sink = 2.0 / 2.5 / 3.0`
- `rubber_sleeve_t = 1.0` + `contact_sink = 2.0`（临界上限）

### 当前名义值的风险
若保持：
- `contact_sink = 2.5`
- `rubber_sleeve_t = 1.5`

则一阶估算：
```text
理论有效压紧 ≈ 4.0 mm
```

这已经明显高于当前 `2–3mm` 目标区间。

---

## v1.2 阶段建议
### 优先验证顺序
1. 先确认是否真的需要连续包胶
2. 若需要，先压缩包胶厚度候选值
3. 再看是否要把默认 `contact_sink` 下调
4. 最后再决定是否需要改局部调节范围

### 当前建议口径
在实测之前，不建议同时默认：
- `rubber_sleeve_t = 1.5`
- `contact_sink = 2.5`

更稳妥的验证起点：
- 方案 A：`rubber_sleeve_t = 0 ~ 0.5`，`contact_sink = 2.5`
- 方案 B：`rubber_sleeve_t = 1.0`，`contact_sink = 2.0`

---

## 通过标准
满足以下全部条件，才算这一项通过：
1. 实测装配后的有效压紧量可稳定落在 `2–3mm`
2. 调节只需短行程微调，不引入新结构方向
3. 球表面不会因过压而显著受损
4. 电机负载和振动不会因过压异常上升

---

## 失败后的修改路径
### 情况 1：压紧量 > 3mm
优先级：
1. 降低包胶厚度
2. 下调 `contact_sink`
3. 必要时修正局部调节槽表达
4. 若仍超限，再进入 `v1.1a`

### 情况 2：压紧量 < 2mm
优先级：
1. 增加包胶厚度
2. 上调 `contact_sink`
3. 用短槽/垫片微调

---

## 结论
这张表的作用不是替代实测，而是先把高风险组合排掉。

按当前 CAD 名义值看，**包胶厚度是 v1.2 最先需要收口的变量之一**，否则很容易把压紧量直接推离目标区间。
