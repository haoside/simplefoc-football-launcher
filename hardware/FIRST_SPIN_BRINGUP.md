# 首版打样与上电 Bring-up 注意事项 v1

## 1. 打样前检查
- ERC / DRC 全绿
- 电源极性、端子方向二次确认
- CAN、Hall、Motor 接口丝印方向明确
- 急停硬切回路确认
- 高低压 / 功率 / 逻辑区丝印区分清楚

## 2. 首次上电顺序
### Host 板
1. 不插传感器、不插执行器
2. 仅上 24V，确认 5V/3.3V 正常
3. 烧录最小固件
4. 检查 UART / CAN 初始化

### Motor Node 板
1. 不接电机，先上电
2. 测 3.3V / VDRV / Driver enable 默认态
3. 检查 MCU 下载与 SWD
4. 检查 Driver fault 是否正常
5. 再接 Hall，不接三相功率输出负载
6. 最后接电机，先低压/限流测试

## 3. 首轮限制
- 电源限流开启
- 不上球
- 先单节点
- 先低 RPM
- 先不开自动供球

## 4. 常见风险点
- Hall 线序错误
- 相线顺序错误
- CAN 终端重复或缺失
- Driver bootstrap / gate 阻值不合适
- Shunt 采样地回流污染 ADC
- 急停只做了软件没做硬切

## 5. 建议首轮验证清单
- Host 供电正常
- Host 串口日志正常
- CAN 通信正常
- Node 下载正常
- Driver 无故障上电
- Hall 读数正常
- 单电机低速闭环正常
- 急停能硬切使能
